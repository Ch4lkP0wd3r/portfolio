PROJECT PHOENIX

Defending Digital Truth · Deepfake Detection · Edge AI · NPU Optimization

A deepfake detection pipeline optimized for Neural Processing Units — eliminating the cloud dependency that makes real-time detection impractical in high-risk environments.

PyTorchOpenCVONNX RuntimeINT8 QuantizationCNN-Transformer
94%
Real-time accuracy · Sub-200ms latency
View Repository

Challenge

By 2026, synthetic media has dissolved the baseline of epistemic trust. Existing detection pipelines are cloud-dependent, latency-heavy, and inaccessible to journalists, activists, and election monitors who need them most. The bottleneck is not intelligence — it is infrastructure.

Action

Implemented INT8 post-training quantization to compress the model footprint without sacrificing accuracy — a tradeoff requiring simultaneous understanding of hardware constraints and the model's loss landscape. Designed a hybrid CNN-Transformer backbone capturing both spatial artifacts and temporal drift across frame sequences, then exported to ONNX Runtime for NPU-targeted deployment.

Impact

94% real-time detection accuracy under edge constraints. Sub-200ms inference latency — viable for live video fact-checking. Detection triggers a confidence score, not an accusation, preserving presumption of innocence while flagging risk. Truth is an engineering problem. This is the solution.

02

CALCPRO

Engineering for the Vulnerable · Mobile Security · AES-256 · Stealth UX

An encrypted evidence vault for domestic violence victims, disguised as a native calculator. The threat model is intimate — security designed adversarially against a human attacker, not just a system.

TypeScriptAES-256Duress PINSection 65BMimicry UX
vault:// authenticate --pin [REDACTED]
[VAULT] AES-256-GCM unlocked
[DURESS] Alternate PIN → silent purge + decoy
[OPSEC] No forensic trace. No network calls.
View Repository

Challenge

Domestic violence victims face a paradox no mainstream safety app has solved: the device is both the lifeline and the evidence. Abusers conduct phone checks. Visible safety apps get deleted — or worse, escalate danger. The threat model is intimate, requiring security posture designed against a human adversary with physical access.

Action

Implemented AES-256-GCM encryption for all stored media. Engineered a mimicry layer — the app's UI, icon metadata, and process name present zero forensic signature distinguishable from a native calculator. Added a duress PIN system: one PIN opens the vault, an alternate PIN purges silently and loads a decoy session — borrowed from intelligence tradecraft, adapted for civilian safety.

Impact

Deliberately omitted a "report" button — because in many cases, reporting is itself dangerous. The app's power is documentation without disclosure. Positioned for NGO partnership and pilot deployment. A proof-of-concept for what the security community calls defensive privacy engineering — a discipline that elite programs are only beginning to formalize.

Challenge

Financial crime detection systems are either inaccessible black boxes or fragile rule-based engines that sophisticated actors trivially bypass. The gap is not data — it is multi-signal reasoning: combining statistical anomalies, graph topology, and geographic risk into a single explainable verdict.

Action

Built a modular intelligence framework evaluating every transaction across 10 forensic layers — transactional behavior, geographic proximity to concealment zones, temporal burst patterns, multi-hop graph layering paths, and 200+ composite risk indicators. Trained a supervised ensemble (XGBoost + LightGBM + Random Forest). All PII hashed with SHA-256 at ingestion — the system flags risk without ever handling identity.

Impact

Every flag accompanied by a forensic narrative explaining the "why" — not just a score, but a human-readable audit trail exportable as structured JSON. Validated with a performance benchmark suite. Research-grade tooling built by a pre-university developer, before any institutional access to compute or mentorship.

Challenge

Agricultural supply chains suffer from a trust collapse at every handoff — exporter to QA agency to importer. Paper certificates are forged, chemical tests are ignored, and importers have no mechanism to verify provenance. The problem is not data — it is the absence of a tamper-proof, globally verifiable trust layer between producers and markets.

Action

Implemented a serverless reference architecture against MOSIP's live Inji stack — acting as a real Postgres Data Provider for VC issuance. Generated W3C-compliant JSON-LD Digital Product Passports using the DigitalProductPassport_v1 schema, deep-linked to the real Inji Web Wallet. Built revocation control, a Farm-to-Fork traceability tree, multilingual support across 5 languages, and an immutable verification audit log.

Impact

Completed 100% of mandatory hackathon deliverables plus all bonus tasks. Deployed live. Demonstrated that a pre-university developer can ship production-grade digital identity infrastructure against a real government sandbox — not a mock, not a simulation, a live integration recognized by MOSIP's international jury.

GUARDIAN BOT

Bridging the Legal-Tech Gap · Civic AI · Cybercrime Triage · NLP · Meridin & Co™

An intelligent conversational system guiding cybercrime victims through the reporting process in real time. Mapped to India's IT Act, IPC sections, and CERT-In protocols. Zero-trust — the bot never stores victim data.

View Repository

Challenge

Cybercrime reporting in India fails at the first step: victims don't know what constitutes a reportable offense, which jurisdiction applies, or what evidence to preserve. Law enforcement receives incomplete reports. Cases collapse before investigation begins. The gap is not legal — it is informational, and informational gaps are engineering problems.

Action

Built Guardian Bot's reasoning layer on a decision tree mapped to India's IT Act provisions, IPC sections, and CERT-In protocols — translating legal complexity into plain-language guidance. Integrated dynamic evidence checklist generation: case-specific lists of what to screenshot, preserve, and timestamp before filing. Zero-trust input — no session data ever stored, so the tool cannot itself become a liability to victims.

Impact

Founded Meridin & Co™ as the civic-tech organization behind the initiative — an operational entity, not a school project. Recruited a cross-functional peer team, validated legal accuracy with student legal aid communities. Meridin & Co™ has a growth roadmap extending into university, where research infrastructure can scale what community resources started.
